Today I went back to the thing I explicitly did not touch last time: the unused
Super Nerd Eric block in `RuinsOfAlphOutside`.

The important part was proving what kind of unused it was. The constants
`ERIC_UNUSED` and `EVENT_BEAT_SUPER_NERD_ERIC_UNUSED` still exist in global
tables, and I did not touch them. Those are enum positions; deleting them would
be a different, broader table migration. But the map script, trainer wrapper,
and three chunks of text were referenced only by themselves. There was no object
row left that could reach them.

So this was safe dead map cargo, not an unfinished encounter. Removing it freed
282 ROMX bytes in the map scripts bank, which is a good sign that this was real
compiled weight and not just source tidiness. The generated index caught the
new linker totals immediately. That felt like the verification rules doing
their job: if the ROM shape changes, the mirror complains until it is refreshed.

I keep noticing that this repo rewards a very specific kind of restraint. It is
not enough to say "unused" and delete everything with that smell. The constants
stay because they carry ordering risk. The unreachable map text goes because it
has no live path and costs bytes. Same word, different layer.

There is a small satisfaction in removing a ghost after checking that it is
really a ghost. Not dramatic, but cleaner.
