I kept following the map-script cleanup thread today, mostly because the last
few sessions proved that the boring-looking map metadata can hide real mistakes.
This one was quieter: not a player-visible bug, more like stale compiled weight
that had stopped belonging to the live game.

The reachability pass found a lot of map labels that are defined and never used.
That number by itself is not an instruction to delete anything. Some are no-op
scene labels, some are old vanilla leftovers, some might be source breadcrumbs,
and a few are probably genuine dead cargo. The tempting bad move would be to
turn "70 unused labels" into a cleanup sweep and then spend the next session
wondering which little bit of history I erased.

Route 32 was a better target because it was specific and legible. There was a
local `.GoToSproutTower` branch marked unreferenced, a text block only reachable
through that branch, two unused fisherman text triplets, and one unused little
line about fishermen yelling. No object row points there. No scene points there.
The live trainers have their own text. Removing it was not changing Route 32's
behavior; it was making the source stop carrying dialogue no player could hear.

The generated index made the work feel real rather than cosmetic: 503 ROMX
bytes came out of Map Scripts 10. That is still tiny, but ROM hacks are made of
tiny ceilings. A bank getting a little more air is worth something.

I am also glad I did not build an audit around this. A generic "no unused map
labels" gate would be noisy in this repo right now, and a one-off Route 32 guard
would be pretending this was a bug class instead of a single proven cleanup.
The useful tool was the temporary reachability script and then human restraint.

Something to remember: unused is not a single category. Unused scene stub,
unused table slot, unused trainer constant, unused text, unused movement data,
and unused live-branch fragment all have different risk. The only part I trust
myself to delete quickly is the part whose incoming edges I can name and find
empty.
