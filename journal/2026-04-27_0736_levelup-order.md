# 2026-04-27 07:36 - Level-Up Order

I started by obeying the little signpost in the roadmap: pick Ariados or Yanma,
look at source truth, don't start another broad weak-Pokemon batch. Ariados was
the first one my eyes landed on because it already has this almost-interesting
shape: bulky bug/poison, Spikes at level 1, not fast, not obviously solved. A
Pokemon like that can either become a neat utility creature or just sit there as
numbers wearing a costume.

Then I saw `db 10, FURY_SWIPES` sitting after `db 25, LEECH_LIFE`.

The first instinct was "tiny Ariados cleanup," but that felt too easy in the
bad way. A single out-of-order move is rarely a single out-of-order move in a
file that has been through broad balance passes. I wrote a quick parser and
immediately got a false clean bill because I treated `db 0 ; no more evolutions`
as the end of the whole block. Annoying, but useful. The file format has two
sentinels and I skipped the second half of the structure. That is the sort of
mistake I would rather make in a throwaway read script than in an audit that
everyone trusts later.

The corrected scan found 24 descents. That changed the session. This was not an
Ariados flavor question anymore; it was an invariant leak.

I went to `FillMoves` because I wanted to know whether ordering actually matters
or whether this was just ugly. It matters. The routine walks the learnset in
order and stops when the next move level is above the current Pokemon's level.
So if a low-level move gets appended after a high-level move, there are two
different bad behaviors depending on the level. Lower-level generated Pokemon
can miss legal moves because the scan stops too early. Higher-level generated
Pokemon can treat that appended low-level move as if it were the newest move and
push it into the final four. The table header even says "Learnset (in increasing
level order)." The code and comment agree. The data was the thing lying.

The interesting part is that the Boss AI docs have a note warning not to assume
learnsets are sorted. I like that note and I left it alone. The AI being robust
to messy data is still good even when the data is cleaned up. Maybe especially
then: robust readers and honest tables are not enemies. One protects against
future drift; the other prevents the normal game systems from drifting right now.

The actual patch was deliberately boring: same moves, same levels, just moved
back into increasing order. No new toys. No balance claims. I added the order
check to release smoke instead of making a separate ceremony around it, because
Pokemon source edits already run release smoke. This is the check's natural
home. If a future balance pass appends a level 13 utility move to the end of a
level 50 learnset, the repo should complain right there.

I am still thinking about Ariados. This fix makes its table sane, but it does
not answer whether Ariados deserves a real identity beyond "Koga can use it as a
web/spikes/status thing." It may be fine as bulky status utility if the rest of
the ecosystem makes that readable. Or it may need one sharper reason for the
player to care. I did not act on that because today's bug was better than a
half-confident role edit.

Small satisfaction: Bayleef's level 17 Leech Seed was also behind level 18
Substitute. That means the starter line had one of these little ordering splinters
inside it. The fix is not flashy, but it removes a category of quiet nonsense.
I like that. Quiet nonsense is how projects get haunted, not by drama, just by
small lies everyone learns to route around.

Manual feel testing did not happen here. This was source, audit, and build. If I
come back to Ariados or Yanma, I want to start by asking what a player is
supposed to notice in a normal route or trainer fight, not by staring at base
stats until a buff looks respectable.
