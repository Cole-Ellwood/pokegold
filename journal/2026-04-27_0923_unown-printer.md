I wandered into the Unown printer because the usual bug nets were quiet and I
wanted to get out of the same few rooms. The first suspicious thing was not a
crash or a bad branch. It was a shrug in source form: "something to do with
Unown printer."

That comment is funny for about one second and then expensive. The function is
doing a very specific old-machine thing: take the 7-by-7 frontpic tile block,
rotate the tile bytes, and reorder the square so the Game Boy Printer receives
the stamp in its expected orientation. The helper named `CountSetBit` was not
counting anything either. It was packing one rotated column byte by walking the
same bit through eight rows.

I did not change the algorithm. I did not want to pretend this was a bug fix.
But this is one of those places where true names matter because the code itself
is already visually dense. `RotateTile` and `PackColumnByte` give the next
reader the right nouns before they have to mentally simulate the carry flag.

The X Unown word looked tempting for a second too. `XXXXX` feels like a
placeholder until you remember this whole table is strange on purpose. I left
that alone. A weird data choice is not automatically a broken data choice, and
the fastest way to make a maintenance branch worse is to "fix" flavor you have
not actually proved wrong.

Small result, but a clean one: one less foggy doorway in an old peripheral path
that almost nobody will touch until the day they absolutely need to understand
it.
