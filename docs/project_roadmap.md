# Current Project Roadmap

## Boss AI Cognition Mode

For Boss AI work, the roadmap should not domesticate the thinking. Record the
wildest fair ideas, the scary boss-feel questions, and the next deep branch, but
keep implementation rows honest about proof, memory, and no-cheat boundaries.

This is the broad work board for future Codex/helper sessions. Use it after
`docs/codex_context.md` and `docs/project_map.md` when deciding what the current
project is and what still needs attention.

Update rule: every session that touches a listed workstream must update its row
when work is completed, started, blocked, deliberately left untouched, or when a
useful idea appears. Keep ideas in `Ideas / next moves`; do not invent new status
values for speculation.

Status legend:

- `UNTOUCHED`: intentionally not started yet.
- `IN_PROGRESS`: started, but not fully verified or finished.
- `STUCK`: blocked by missing input, missing tooling, unclear source truth, or a
  required manual/runtime step.
- `COMPLETE`: finished and verified enough for the evidence listed.

Source-truth rule: this roadmap is an index of current work, not an authority
over source. If it conflicts with source files, linker outputs, or generated
audits, trust the source truth and update this file.

Felt-promise rule: every workstream must name the experience it protects. Beauty
is first evidence that the underlying order is right, then a force that makes
future helpers trust, linger, and protect the work. If a change is technically
correct but makes the hack feel cheap, arbitrary, noisy, dead, or less like fair
hard Pokemon Gold, treat that as a real defect and record it here.

First-Playthrough Promise: roadmap work protects the feeling of being small in
Johto again. The hack should restore uncertainty for a veteran player: bosses
can beat you without cheating, forgotten Pokemon can surprise you, and QoL
respects the journey instead of dissolving it.

Urgency order is top-to-bottom: `DO NOW`, `NEXT`, `SOON`, `WAIT`,
`REFERENCE`. Urgency is separate from status. A complete row can still be
important, but it belongs lower once it is no longer the next decision.
