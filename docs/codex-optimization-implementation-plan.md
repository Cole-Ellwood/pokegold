## Ground Rules

- Keep patches small. Do not mix comment-only cleanup, local no-op removal, table ABI work, and save-format work in one patch.
- Before each patch, re-read the linked findings and run a final `rg` for each label, constant, macro, table row, and text label being changed.
- For ASM behavior changes, build after each batch. If a batch touches battle math, run the damage oracle or the named battle audit; do not hand-compute expected damage.
- For `ram/`, `hram`, `sram`, or save-version changes, keep layout unchanged unless the whole patch is explicitly a save-format patch with `SAVE_FORMAT_VERSION` handling and `python tools/audit/check_save_format_version.py`.
- For farcall, predef, special, ROM0, or table-indexed changes, document before/after register clobbers or numeric ABI expectations in the patch.
- Update the checkboxes here as patches land. Put the commit hash or PR name beside the item once verified.
