# Worked Example: Prediction-Minimizing Pressure

Source: Stathakis, A Greek RMT:
https://www.smogon.com/forums/threads/a-greek-rmt.53728/

Format: DPP OU suspect ladder. This is a strategic source study, not GSC or
romhack mechanics proof.

## Source Lesson

The source argues that offense does not need to depend on flashy prediction if
the team is built and piloted so that normal defensive answers all concede
progress. The key pattern is overlap: several attackers pressure the same
limited defensive answers, and one attacker can be spent to weaken a blocker so
the next attacker inherits a better board.

That idea transfers better than the specific team or DPP mechanics.

## Position Pattern

```text
Our side:
Several pieces pressure the same blocker family.
One piece can be spent or damaged if it makes the blocker weak enough.
The intended closer is preserved for after the blocker has been priced.

Opponent side:
The opponent has one or two stabilizers that must answer several threats.
If they do not use the stabilizer, the active threat makes immediate progress.
If they do use it, it takes the damage/status/trade the route wanted.
```

## Candidate Move Classes

Best when available:

- Make the move that improves the route against both the stay-in branch and the
  expected defensive answer.
- Use the active attacker to damage, status, or force the shared blocker before
  preserving it becomes more important than route pressure.
- Save the true cleaner until the blocker map is known and weakened.

Acceptable:

- Sacrifice a pressure piece if the sacrifice puts the shared blocker into the
  cleaner's range and the opponent cannot set up for free afterward.
- Switch only when staying in gives the opponent a free conversion route or
  spends an irreplaceable piece.

Wrong:

- Making a narrow prediction when a broad pressure move advances the route
  against multiple branches.
- Preserving every attacker and letting the shared blocker stay healthy.
- Sacrificing without naming which teammate inherits the improved board.

## Rule

Before making a prediction-heavy move, fill this out:

```text
Shared blocker:
Threats that pressure it:
What happens if opponent stays:
What happens if opponent uses the blocker:
Piece that inherits progress:
Free setup branch we must not allow:
```

If the same move improves both the stay branch and the blocker branch, prefer
that move over a stylish read. Prediction is strongest when the opponent's
reasonable responses are already constrained.

## Boss-Battle Transfer

Against Gym Leader Lab bosses, this becomes route forcing rather than human
bluffing. The boss may not preserve pieces like a tournament player, but the
same structure still matters:

1. Which boss Pokemon must answer several of our routes?
2. Can the current move punish both the active and that answer?
3. If the boss does not use the answer, what progress do we get instead?
4. If the boss does use it, which player piece inherits the damage/status?
5. Does staying in give the boss setup, weather, hazards, sleep, or recovery
   that is worse than the pressure gained?

## Caveat

The source is intentionally extreme about not switching and avoiding choice
lock. Do not copy that literally. In GSC and Gym Leader Lab, switching can be
the best move when it preserves an irreplaceable answer, resets a bad active
state, or avoids donating a setup route. Transfer the principle, not the habit:
reduce guesswork by choosing route-forcing moves.
