# Heuristic: Branch Punish Ranking

Status: live tiny card.

Use when: the answer names a receiver, absorber, immunity, setup branch, or
counter-pivot, but the top move may still only beat the active target.

Rule: a named branch must change the ranking.

Ask:
- What action beats the branch: coverage, status, phaze, setup, switch, cash-out?
- Does active pressure already hit the branch hard enough?
- After our handoff, does the new owner invite a counter-owner that must be
  hit before generic status/setup?
- When our Electric pressures Water/Flying, does Hidden Power beat the
  Ground/Steel counter-owner better than STAB into only the active target?
- Does revealed setup, coverage, or cash-out beat the branch before a switch
  handoff can stabilize it?
- If the branch is a low sleeper or support piece, does Self-Destruct,
  Explosion, or a forced sack beat our active before damage pays off?
- If the named branch is the special wall, Electric blocker, or last breaker
  that stops the route, is cash-out the punish rather than more chip?
- For Explosion users, is this a Ghost branch, resist branch, or true-blocker
  branch, and does post-boom free entry beat the preserving line?
- In a boost/pass mirror, does attacking the passer, boosting again, or passing
  to the receiver punish the next board after the immediate hit?
- Against AgilityPass, does Reflect/status/phaze punish the receiver branch
  better than damage into the passer?
- After a boosted passer faces RestTalk, does the branch punish require Baton
  Pass to a receiver rather than another attack into the active target?
- With Spikes up and a sleeping wall active, is the handoff branch a spinner or
  phazer that must be punished before generic setup?
- If Perish Song forced a switch, is the branch punish to trap the receiver or
  cover the immediate attack with Destiny Bond instead of attacking for chip?
- Is the branch revealed, strong-prior, or possible-only?

Promote the branch-punish when it wins the next board. Keep active pressure
when it beats both active and branch.

Do not name a branch and then leave its punish below a generic safe move.
Generic status/setup is not the punish unless it beats the named next owner or
counter-owner.
Damage into a low cash-out user is not a branch punish when the trade removes
the route piece; the punish is the resist, Ghost, low-value sack, or exact KO
that leaves the next owner named.
Damage into a revealed package is not branch punish if the cash-out, status, or
controlled-sack branch still wins the next board.
In explosion-stack endgames, preserving the exploder is not branch punish when
the named blocker can be traded for a clean closer or free-entry owner.
Explosion branch punish is branch-specific. A Ghost branch usually makes
coverage or handoff the punish, not boom. A resist branch can still be punished
by boom only when resisted chip plus action denial and free entry creates a
named route owner. If ordinary damage covers the same branch while preserving
boom, rank ordinary damage first.
Boosting is not progress by itself; it is progress only if the pass receiver or
current attacker survives the reply and wins the next board.
Boosted damage is not branch punish when a revealed or strong-prior Sleep Talk
roll removes the passer first. The punish is the receiver, phazer answer, or
threshold attack that survives the roll.
Do not overguard a possible cash-out when the current move gets the exact KO
or removes the blocker before it can reset, phaze, or trade.
