Battle payout still masks all three bits, has a special comparison for a combined half-saving value, and keeps separate "sent some", "sent half", and "sent all" text pointers.

This may be intentional compatibility for old save states or debug-set flags, but future agents cannot tell from the code. A cleanup that trusts the WRAM comment would delete live battle branches/text; a battle-flow edit that trusts the table would preserve modes that no UI can select.

Suggested Change

Choose an explicit policy. If only "some money" is supported, remove the half/all payout branches and text rows in a save-format-risk aware patch, and keep old save-bit migration behavior in mind. If half/all are intentionally accepted for compatibility, change the WRAM/constants comments to say "not selectable by current UI; battle payout still honors legacy bit patterns" and add a small signpost near .SentToMomTexts.
